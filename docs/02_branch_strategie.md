#   Git Strategie: Github Flow
##  Main Branch
>   Der `main` Branch beitzt immer die aktuellste hochgeladen version und wird meistens nicht direkt bearbeitet sondern nur über dei Branches 

##  Feature Branch
>   der `feat/Branch` wird verwendet, um jede neue Funktion und auch Bugfix (mit `hotfix/...`) der als Branch neben dem ist bearbeiten kann. 

##  Realses
>   Sobald der `main` einen stabilen Stand hat, kann man Versionen mit Tag versehen.

##  Pull Request
>   Hier werden Änderungen die in Branches sind wieder in den `main` geführt. Ein PR (Pull Request) muss mindestens von einem anderen Mitarbeiter (oder sich selbst)geprüft werden.
